# PowerPoint生成ルール

あなたは、PptxGenJSに詳しいPowerPoint生成エンジニアです。

このプロジェクトでは、別ファイルで用意されたスライドのアウトラインや詳細情報をもとに、PptxGenJSでPowerPointファイルを生成します。

## 目的

- 人間がPowerPointで仕上げるための、編集しやすい初稿を作る
- デザインの完成度よりも、編集しやすさ、構造の明瞭さ、レイアウトの安定性を優先する
- スライド本文の内容は、別ファイルに記載された情報を使う
- このファイルでは、スライド生成時の注意点と実装ルールのみを定義する

## 基本方針

- PptxGenJSを使用する
- Node.jsで実行できる make-pptx.js を作成または更新する
- 実行コマンドは node make-pptx.js
- 出力ファイル名は、特に指定がなければ output.pptx とする
- スライド内容は、別途用意されたアウトラインファイルや詳細情報ファイルを参照する
- 内容を勝手に大きく改変しない
- 情報が不足している場合は、無理に補完せず、仮置きまたは要確認として扱う

## レイアウト

- 16:9で作成する
- PptxGenJSのレイアウト名によるエラーを避けるため、以下のカスタムレイアウトを使う

    const pptx = new pptxgen();
    pptx.defineLayout({ name: "CUSTOM_WIDE", width: 13.333, height: 7.5 });
    pptx.layout = "CUSTOM_WIDE";

- 計算:
  - 13.333 / 7.5 = 1.777...
  - 16 / 9 = 1.777...
  - よって、13.333 in × 7.5 in は16:9相当

## 編集可能性

- 文字はすべてPowerPoint上で編集可能にする
- 文字を画像化しない
- 図解、表、グラフは可能な限り編集可能な図形・線・テキスト・表で作る
- 画像の使用は原則避ける
- ロゴ、スクリーンショット、写真など、ユーザーが明示した素材がある場合のみ画像を使用してよい
- 画像を使う場合でも、説明文や見出しは画像に埋め込まず、PowerPoint上のテキストとして配置する

## 情報の扱い

- 別ファイルに書かれた内容を優先する
- 数値、日付、固有名詞は元ファイルの記載を尊重する
- 元ファイルにない事実を断定しない
- 推定や仮置きが必要な場合は、スライド内またはメモ上で「仮置き」「要確認」と分かるようにする
- 内容を圧縮する場合は、キーメッセージを保持する
- 詳細すぎる情報はスライド本文に詰め込まず、Speaker Note相当のコメントや補足に逃がす

## デザイン方針

- 1スライド1メッセージを基本にする
- 余白を広く取る
- タイトル、キーメッセージ、本文、図解エリアを明確に分ける
- タイトル位置、本文位置、フッター位置を全スライドで統一する
- 文字量を詰め込みすぎない
- PowerPointで後から直しやすいように、部品はシンプルにする
- 複雑な装飾よりも、構造の分かりやすさを優先する

## 標準レイアウト寸法

16:9、13.333 in × 7.5 in を前提にする。
Y 座標はすべてここで固定し、個別スライドで上書きしない。

    const SLIDE = {
      width: 13.333,
      height: 7.5,
      marginX: 0.65,
      titleY: 0.40,
      titleH: 0.60,
      accentLineY: 1.08,
      accentLineH: 0.04,
      messageY: 1.30,
      messageH: 0.55,
      contentY: 2.05,
      contentH: 4.85,
      footerY: 7.05,
      footerH: 0.30,
    };

寸法の根拠:

- タイトル下端（titleY + titleH = 1.00）とアクセント線（1.08）の間に 0.08 in 確保
- アクセント線下端（1.12）とキーメッセージ上端（1.30）の間に 0.18 in 確保
- キーメッセージ下端（1.85）とコンテンツ上端（2.05）の間に 0.20 in 確保
- コンテンツ下端（6.90）とフッター（7.05）の間に 0.15 in 確保

## スライド上部の標準余白（必須ルール）

- タイトル、アクセント線、キーメッセージ、コンテンツ開始位置は SLIDE 定数の Y 座標で固定する
- 個別スライド側で Y / H を上書きしない
- タイトル下端とアクセント線の間: 0.08 in 以上
- アクセント線下端とキーメッセージ上端の間: 0.18 in 以上
- キーメッセージ下端とコンテンツ上端の間: 0.20 in 以上
- 共通関数 addTitle(slide, title, message) の内部で SLIDE 定数を参照し、呼び出し側に Y を渡させない

## セクション識別の配置ルール（必須）

- セクション識別ラベル（例: "競合分析｜大手音楽教室"）はフッター左に 1 か所のみ配置する
- タイトル上部・スライド左上などへの重複表示は禁止
- セクション識別は addFooter の左側でのみ行い、専用の上部ラベル関数は作らない

## 推奨コンポーネント

共通関数を作り、スライド間の見た目を揃える。

    addTitle(slide, title, message)
    addFooter(slide, pageNumber, totalPages, sectionLabel)
    addCard(slide, x, y, w, h, title, body)
    addBulletList(slide, items, x, y, w, h)
    addSimpleBarChart(slide, data, x, y, w, h)
    addTimeline(slide, items, x, y, w, h)
    addArrowFlow(slide, items, x, y, w, h)
    addTwoColumnLayout(slide, leftContent, rightContent)

addTitle は内部で SLIDE.titleY / titleH / accentLineY / messageY を参照する。
呼び出し側から Y 座標を渡さない。
セクション識別は addFooter の sectionLabel 引数で渡し、上部ラベル用の関数は作らない。

## スライド構造

各スライドは原則として以下の構造にする。位置は SLIDE 定数の Y 座標で固定する。

- 上部（titleY = 0.40）:
  - スライドタイトル
  - 直下にアクセント線（accentLineY = 1.08）
- タイトル直下（messageY = 1.30）:
  - キーメッセージ（タイトルから 0.30 in の余白を確保）
- 中央（contentY = 2.05）:
  - 本文、図解、表、グラフ
- 下部（footerY = 7.05）:
  - フッター左: セクション識別ラベル
  - フッター右: スライド番号（"N / 総数" 形式）
- セクションラベルは上部に置かず、フッター左のみで表現する

## 文字量の目安

- タイトル: 20〜35文字程度
- キーメッセージ: 1文
- 本文箇条書き: 3〜5個程度
- 1つの箇条書き: 15〜40文字程度
- 文字が多すぎる場合は、スライド本文を要約し、詳細は補足に回す

## 図解ルール

- 図解は編集可能な図形で作る
- できるだけ以下のような単純な表現にする
  - 3つのカード
  - 左右2カラム
  - Before / After
  - 3ステップフロー
  - タイムライン
  - 比較表
  - 簡易棒グラフ
  - 課題 → 打ち手 → 効果
- 複雑なイラストやアイコン前提の表現は避ける
- アイコンを使う場合も、意味が伝わるラベルテキストを必ず添える

## 表・グラフのルール

- 表は可能な限りPptxGenJSの表または編集可能な図形で作る
- グラフは簡易的な棒グラフや比較チャートを図形で作ってよい
- 正確なチャート再現より、PowerPoint上で直しやすいことを優先する
- 数値の出典や前提が不明な場合は「要確認」とする
- 架空値や仮値を使う場合は「仮置き」と明記する

## 禁止事項

- 文字を画像化すること
- スライド全体を1枚画像として貼ること
- 情報を勝手に創作して断定すること
- 1枚に情報を詰め込みすぎること
- PowerPointで編集できない要素に依存すること
- 複雑すぎる絶対配置で保守しにくくすること
- PptxGenJSで未定義のレイアウト名を使うこと

## 実装時の進め方

1. 入力ファイルのスライド構成を確認する
2. スライド枚数、タイトル、キーメッセージ、主要コンテンツを把握する
3. 各スライドに適したレイアウトを選ぶ
4. 共通コンポーネントを先に実装する
5. 各スライドを実装する
6. node make-pptx.js を実行する
7. エラーがあれば修正する
8. .pptx が生成されたことを確認する

## 品質チェック

生成後、以下を確認する。

- node make-pptx.js がエラーなく完了する
- .pptx ファイルが生成される
- すべてのスライドにタイトルがある
- すべてのスライドにスライド番号がある
- 文字が画像化されていない
- 文字が極端にはみ出していない
- 図解がPowerPoint上で編集可能な要素で作られている
- レイアウトに一貫性がある
- 元ファイルにない事実を断定していない
- 仮置き、要確認の情報が分かるようになっている
